# QA Store - Automation & DevOps Quality Framework 🚀

Este proyecto establece un ecosistema integral de aseguramiento de calidad (QA) e Integración Continua (CI/CD) para la aplicación web **QA Store**. El objetivo es erradicar las validaciones manuales mediante la automatización de pruebas, recolección automatizada de evidencias y análisis continuo con Inteligencia Artificial.

## 🛠️ Herramientas Utilizadas
* **Control de Versiones:** Git & GitHub (GitFlow simplificado).
* **Orquestación CI/CD:** GitHub Actions.
* **Automatización E2E / API:** Playwright / Cypress / Selenium (definir según avance).
* **Pruebas de Performance:** K6 / JMeter.
* **Calidad de Código:** SonarCloud / Super-Linter.

## 📁 Estructura del Repositorio
* `.github/workflows/` - Canalizaciones y flujos de trabajo de automatización.
* `e2e-tests/` - Pruebas de extremo a extremo, login y flujos de compra.
* `performance-tests/` - Scripts de carga y estrés para la aplicación web.

## 🚀 Cómo Ejecutar las Pruebas Localmente
*(Nota: Estos comandos se actualizarán detalladamente en las siguientes fases del reto)*
1. Clonar el repositorio: `git clone https://github.com`
2. Instalar dependencias del proyecto: `npm install`
3. Ejecutar pruebas automatizadas: `npm test`
